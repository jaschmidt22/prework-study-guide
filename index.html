HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user
  • The title and meta elements are imbedded inside of the head. These provide information about a website to the browser and search engines.
  • Inside the body h1, h2, h3, etc. represent a heading.
  • Also in the body p represents a paragraph, ul is used for unordered lists, ol for ordered lists, and li for list items
  • The img element contains information about images that are displayed.
  • The br element creates a line or empty space, or a line break, between two blocks of content

CSS

  • A margin indicated how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Inline CSS is used to change a particular page element only
  • Internal CSS rules are placed in the head section of the HTML file and apply to that page only
  • External CSS allows developers to keep all their CSS rules in a seperate file, whixh makes changes easier
  • When using an external style sheet we link the file using a link element placed in the HTML's head
  • A selector defines the element or attributes to which the rules, or declarations, will apply
  • Declarations contain the CSS property we want to apply and the value of that property

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • git log: show the commit history for the currently active branch
  • git pull: fetch and merge any commits from the tracking remote branches
  • git is great for tracking changes locally on your own computer
  • GitHub is a cloud-based hosting service that allows us to keep track of our projects inside repositories that are stored remotely
